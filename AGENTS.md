# AGENTS.md — ProCoursing Stats

Проект: агрегатор статистики собак по результатам procoursing.ru (курсинг/БЗМП/бега).
Полная спецификация — `README.md`, схема БД — `schema.sql`. Прочитай оба перед началом работы.

## Зафиксированные решения (не предлагать пересмотреть без явного запроса)

- Охват: все породы (с фильтром в UI), весь архив 2015–2026 (с фильтром по
  году, по умолчанию открыт текущий год).
- Топ — ДВА отдельных рейтинга, не один: по местам (медальный зачёт:
  золото→серебро→бронза) и по очкам (лучший результат + средний + число
  стартов рядом). Не сводить в одну формулу.
- Родословные — переход на внешний сайт по клику, мы их сами не храним и
  не парсим PDF-каталоги. Какой именно сайт — пока открытый вопрос,
  спроси меня, если дойдёшь до этой части.
- Стек: Cloudflare Pages + Worker + D1, как в моём другом проекте SW Forge.
  Бэкафилл истории — разовый Node-скрипт локально, текущий год —
  инкремент по Cron Trigger.

## Известные риски / куда смотреть внимательно

- Сайт-источник в кодировке windows-1251, без явного charset в заголовках.
  Скрапер ОБЯЗАН декодировать вручную (`lib/fetch-win1251.mjs`, iconv-lite),
  иначе кириллица теряется безвозвратно (не мусор — реальная потеря байт).
- `scripts/scrape-year-index.mjs` — написан по структуре, которую я видел
  через инструмент с битой кодировкой (но точными URL). Структура таблицы
  (порядок td) — best guess, проверь на реальном HTML первым делом.
- `scripts/parse-results-coursing.mjs` — есть рабочий тест
  (`scripts/test-parser.mjs`, `node scripts/test-parser.mjs`), но он на
  синтетических данных. ОБЯЗАТЕЛЬНО прогони на 5-10 реальных страницах
  разных лет, прежде чем доверять результатам массово. Особое внимание:
  - перенос клички через "/" на следующую строку (уже обработано, но
    проверь на других годах — могла измениться вёрстка)
  - статусы "снят"/"дисквалифицирован"/"не финишировал" — сейчас
    определяются эвристикой "нет bold-итога = что-то нестандартное",
    раз я не могу прочитать кириллицу для точного маппинга — после
    починки кодировки можно сделать точнее через ключевые слова.
- БЗМП и бега (Racing) — отдельные форматы страниц, парсер для них ещё не
  написан вообще. Не предполагай, что формат курсинга подойдёт.

## Рабочий процесс

1. Перед любым изменением парсера — прогони `node scripts/test-parser.mjs`.
   После изменения — прогони снова, тест должен остаться зелёным (или
   обнови тест, если изменилась ожидаемая структура, и объясни почему).
2. Не трогай зафиксированные решения выше без явного запроса.
3. При парсинге реальных страниц — сохраняй `raw_text` всегда, даже когда
   уверен в разборе. Это страховка от тихой потери данных.
4. Будь вежлив к сайту-источнику: пауза между запросами (уже есть в
   `lib/fetch-win1251.mjs` / `sleep()`), не параллелить запросы при бэкафилле.
