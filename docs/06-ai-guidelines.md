# 06. AI Guidelines — Рекомендации для ИИ-агента

## Общие правила

1. **Всегда читай AGENTS.md перед началом работы** — там зафиксированные решения, которые нельзя менять без явного запроса.
2. **Перед изменением парсера — прогони тест** — `node scripts/test-parser.mjs` должен быть зелёным.
3. **Не трогай зафиксированные решения** — два отдельных топа, все породы, весь архив, родословные на внешний сайт.
4. **Сохраняй raw_text всегда** — это страховка от потери данных при ошибках парсинга.

## Работа с кодировкой

⚠️ **Критически важно:** Сайт procoursing.ru использует windows-1251 без charset в заголовках.

- Используй `iconv-lite` для декодирования
- НЕ доверяй `fetch().text()` — декодируй из байт вручную
- Пример в `fetch-win1251.mjs` (нужно исправить см. TECHNICAL_PLAN.md)

## Тестирование парсеров

1. **Синтетические тесты:** `node scripts/test-parser.mjs`
2. **Реальные данные:** Прогони на 5-10 страницах разных лет
3. **Проверяй:**
   - Каталожные номера
   - Клички (рус/лат)
   - Баллы судей
   - Итоги
   - Титулы
   - Статусы (снят, дисквалифицирован, не финишировал)

## Разбивка больших файлов

Если файл превышает 2000 строк:

1. **Вынеси константы** в отдельный файл `constants.mjs`
2. **Вынеси утилиты** в `lib/utils.mjs`
3. **Разбей по функциональности:**
   - Парсеры: `parse-results-coursing.mjs`, `parse-results-bzmp.mjs`, `parse-results-racing.mjs`
   - Скраперы: `scrape-year-index.mjs`, `scrape-event-details.mjs`
   - Загрузчики: `load-to-d1.mjs`
4. **Используй модульную структуру:**
   ```
   lib/
     fetch-win1251.mjs
     utils.mjs
     constants.mjs
   scripts/
     scrape/
       scrape-year-index.mjs
       scrape-event-details.mjs
     parse/
       parse-results-coursing.mjs
       parse-results-bzmp.mjs
       parse-results-racing.mjs
     load/
       load-to-d1.mjs
   ```

## Вежливость к сайту-источнику

- Добавляй паузу между запросами (функция `sleep()` в `fetch-win1251.mjs`)
- Не делай параллельные запросы при бэкафилле
- Проверь `robots.txt` сайта

## Работа с памятью

Создавай воспоминания для:
- Зафиксированных архитектурных решений
- Открытых вопросов и рисков
- Важных паттернов кода
- Изменений в структуре сайта

## Порядок работы

1. Прочитай `AGENTS.md` и `README.md`
2. Проверь `docs/` для контекста
3. Перед изменением — прогони тесты
4. После изменения — обнови документацию
5. Создай/обнови воспоминания для важных решений
