# Changes - 2026-07-04

## Admin Interface for Historical Data

### Backend Changes

#### New Admin API Endpoints (`backend/src/routes/admin.ts`)
- `GET /api/admin/events` - List all events before 2026 (historical data)
- `GET /api/admin/events/:id/results` - Get results for a specific event
- `PUT /api/admin/events/:id` - Update event details (title, location, judges, results_url)
- `PUT /api/admin/results/:id` - Update result (placement, total_score, qualification, vc, status)
- `POST /api/admin/results` - Create new result
- `DELETE /api/admin/results/:id` - Delete result

All endpoints protected by `X-Admin-Token` header with `ADMIN_API_TOKEN` from Cloudflare Workers secrets.

## OCR Experiment (Not Used in Production)

### Installed Dependencies
- `tesseract.js` - OCR library for JavaScript
- `sharp` - Image processing library

### Created Scripts
- `backend/scripts/download-2015-images.ts` - Download 2015 event images from procoursing.ru
- `backend/scripts/test-ocr.ts` - Test OCR on single image
- `backend/scripts/test-ocr-preprocess.ts` - Test OCR with image preprocessing
- `backend/scripts/test-ocr-segment.ts` - Test OCR with row segmentation
- `backend/scripts/parse-ocr-results.ts` - Parse OCR text into structured data

### Results
- OCR quality insufficient for automatic data extraction (~50% confidence)
- Names and numbers consistently misrecognized
- Decision: Manual entry via admin interface instead of OCR

## Usage

### Admin Interface
1. Start local backend: `npx wrangler dev backend/src/worker.ts --remote --port 8787`
2. Start frontend: `cd frontend && npm run dev`
3. Open `http://localhost:5173/admin`
4. Enter `ADMIN_API_TOKEN` (from Cloudflare Workers secrets)
5. Edit historical events (year < 2026)

### Admin Token Management
- Token stored in localStorage as `admin_token`
- Verified via API on each login
- Removed on 401 error or logout
- For local development without token: backend allows access if `ADMIN_API_TOKEN` not set in environment
