### 11. Автоматическое обновление сайта для пользователей
**Проблема:** При деплое новой версии сайта пользователи могут видеть старую версию из-за кэширования браузера. Нет механизма автоматического обновления.

**Требование:** Обеспечить автоматическое обновление сайта у пользователей при наличии новой версии без необходимости вручную обновлять страницу.

**Возможные решения:**
1. **Service Worker (PWA):**
   - Проверять обновления при загрузке сайта
   - Уведомлять пользователя о наличии обновления
   - Автоматически обновлять при следующей загрузке
   - Использовать workbox для управления кэшем

2. **Cache-Control заголовки:**
   - Настроить правильные заголовки на Cloudflare Pages
   - Использовать ETag для валидации кэша
   - Установить короткое время кэширования для HTML
   - Долгое кэширование для статических файлов с версионированием

3. **Версионирование файлов:**
   - Vite уже использует хэши в именах файлов
   - Добавить версию в manifest.json для PWA
   - Использовать query parameters для принудительного обновления

4. **Автоматическое обновление страницы:**
   - Polling для проверки обновлений (раз в 5-10 минут)
   - WebSocket для мгновенного уведомления
   - Показывать баннер "Доступна новая версия" с кнопкой обновления

**Приоритет:** Средний

**Описание:** Добавить возможность перейти на профиль собаки на breedarchive.com из рейтинга.

**Анализ:**
- Прямая ссылка на профиль автоматически **невозможна** — URL содержит случайный UUID из их базы, который нельзя вычислить из клички
- Пример: `https://saluki.breedarchive.com/animal/view/emul-de-gepard-gelila-al-rawda-699f7593-397c-4b71-89a7-09e352f30907`

**Рабочий компромисс:** Ссылка на поиск с предзаполненной латинской частью клички — человеку нужно лишь кликнуть на нужный результат поиска.

**Что нужно проверить перед реализацией:**
1. Точный параметр URL в форме поиска на `saluki.breedarchive.com/animal/index` (поле "Registered name") — нужно открыть страницу и посмотреть имя параметра в коде формы
2. Поддомены под каждую породу свои (`saluki.breedarchive.com`, `sloughi.breedarchive.com` — подтверждены). Нужно составить маппинг `порода → поддомен` и обрабатывать случай когда поддомена для породы нет (не показывать кнопку или показывать общую ссылку)

**Файл для реализации:** `frontend/src/pages/DogProfile.tsx`, возможно компонент `DogName` (пункт 17)
