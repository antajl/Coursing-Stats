## Два вида замеров (не пересекаются)

На полигоне Донино ведутся **две независимые серии замеров**. Это разные дисциплины, разные Google-таблицы, разные таблицы и **разная статистика на сайте**. Данные одной серии **нельзя** подставлять в другую (в т.ч. пересчитывать время из км/ч замера скорости и наоборот).

### 1. Замер скорости

**Что это:** на собаку надевают ошейник с датчиком; во время **курсинга** фиксируют **скорость в км/ч**. Это не забег на ровные 350 м с секундомером.

| | |
|---|---|
| **Смысл** | Личный рекорд скорости на курсинге |
| **Первичная величина** | км/ч |
| **Google Sheet** | [1NTiY3HXZ…](https://docs.google.com/spreadsheets/d/1NTiY3HXZIkXE8xTeXZESgMKaZsEXunmcWhTfhhkoKyE) |
| **Runtime** | `data/v1/donino/speed_records.json` |
| **UI** | Колонка «Замер» на `/speed-records`; режим **Записи / Статистика** — в меню «Донино» (`?view=stats`) |

### 2. Бега борзых 350 м

**Что это:** отдельная **ровная трасса ровно 350 метров**, время — **секундомером**.

| | |
|---|---|
| **Смысл** | Зачёт на фиксированной дистанции 350 м |
| **Первичная величина** | секунды |
| **Google Sheet** | [1hpdA8vl…](https://docs.google.com/spreadsheets/d/1hpdA8vlIfeECgpnPvuk5xfezPsdUh1EXULjeATAF9dw) |
| **Runtime** | `data/v1/donino/coursing_records.json` |
| **UI** | Та же страница, колонка «Бега 350 м» |

**Скорость км/ч в статистике 350 м** — только **вспомогательный пересчёт**: `1260 / время_сек`. В таблице курсинга её нет как исходного поля.

### Одна собака в обеих дисциплинах

Одна и та же собака (**кличка + порода**) может быть в обоих листах. Метрики **не объединяются**: лучший км/ч и лучшие секунды — два разных рекорда; на сайте — два независимых блока.

### Правила для кода и UI

1. **Не смешивать источники:** `speed_records` ↔ `coursing_records` только на уровне профиля (две карточки), не в одной формуле «лучшее время».
2. **Не выводить время 350 м из max(км/ч)** таблицы замеров — типичная ошибка (напр. 19,69 с из 64 км/ч вместо реального зачёта).
3. **Фильтры статистики:** в «Статистика» у 350 м — только `coursing_records`; у замера — только `speed_records`.
4. **Пол** в статистике 350 м подтягивается из `speed_records` по `name+breed` (в листе курсинга пола нет) — только для группировки.
5. **Одна собака в обоих листах** — нормально; два независимых блока, не один сводный рекорд.

## Обзор (техническая сводка)

| Назначение | Runtime JSON | Единица |
|------------|--------------|---------|
| **Замер скорости** | `donino/speed_records.json` | км/ч |
| **Бега 350 м** | `donino/coursing_records.json` | сек |

**Обновление обеих таблиц:** `scripts/update-donino.bat` / `npm run export-donino` → commit + push.  
**Cron** `update-speed-records.yml`: **только** speed, не coursing.

## UI (ключевые факты)

**Маршрут:** `/speed-records` — одна страница, **две колонки** (Замер | Бега 350 м).  
**Режим** Записи | Статистика — в шапке «Донино» (`?view=stats`), не дублировать сегмент в тулбаре.

| Компонент | Назначение |
|-----------|------------|
| `useSpeedRecordsPage.ts` | State, фильтры, сортировка |
| `DoninoRecordsColumns.tsx` | Список записей |
| `DoninoStatsColumns.tsx` | Статистика / группы |
| `DoninoPageToolbar.tsx` | Фильтры, Excel-экспорт |

**Фильтры:** пол и км/ч — только «Замер»; сек — только «Бега 350 м».  
**Среднее** в группе — при ≥3 зачётах (`MIN_SAMPLES_FOR_AVG`).

### Профили `/donino-dog/:name/:breed`

| Блок | Источник |
|------|----------|
| Замер скорости | `speed_records` (лучший + `history`) |
| Бега 350 м | `coursing_records` |

История 350 м: `expandCoursingTimeline` в `recordDates.ts` — не смешивать с км/ч.

Атрибуция: `DoninoAttribution` → [runningdog.ru](https://runningdog.ru/).

## Ключевые концепции

| Операция | Ключ |
|----------|------|
| Дедупликация speed | `name + breed + sex + date + speed` |
| История / список UI | `name + breed` |
| Дедуп coursing | `name + breed + time_seconds + date` |

**Дедупликация** — убрать точные дубли из разных листов.  
**История** — разные даты одной собаки для прогресса.
