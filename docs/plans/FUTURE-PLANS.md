```

```

**Рекомендации Vite:**
- Использовать `dynamic import()` для code-splitting
- Настроить `build.rollupOptions.output.codeSplitting`
- Увеличить `build.chunkSizeWarningLimit`

**Возможные решения:**
1. **Code-splitting по страницам:**
   ```javascript
   const Judges = lazy(() => import('./pages/Judges'))
   const JudgeDetail = lazy(() => import('./pages/JudgeDetail'))
   const SpeedRecords = lazy(() => import('./pages/SpeedRecords'))
   // и т.д.
   ```

2. **Разделение компонентов:**
   - Вынести тяжелые компоненты в отдельные файлы
   - Использовать lazy loading для больших таблиц

3. **Настройка rollupOptions:**
   ```javascript
   build: {
     rollupOptions: {
       output: {
         manualChunks: {
           'react-vendor': ['react', 'react-dom', 'react-router-dom'],
           'ui-components': [...],
           'pages-judges': ['./pages/Judges', './pages/JudgeDetail'],
           'pages-speed': ['./pages/SpeedRecords', './pages/SpeedRecordsStats'],
         }
       }
     }
   }
   ```

## Парсинг результатов

### 4. Парсинг рейсинга (Racing)
**Проблема:** Формат результатов рейсинга существенно отличается от курсинга и БЗМП.

**Текущий формат рейсинга:**
- 18 колонок (vs 25 в курсинге/БЗМП)
- Нет rowspan=2
- Метрики: время и скорость (не судейские оценки)
- До 3 забегов на собаку
- Дистанция указана в метрах
- Попоны часто не указаны ("-")

**Пример структуры:**
```
Место | № | Порода | Класс | Пол | Кличка | Дистанция (м) | 
Забег 1 | Попона | Время 1 | Скорость | 
Забег 2 | Попона | Время 2 | Скорость |
Забег 3 | Попона | Время 3 | Скорость |
ВС | Титул(ы)
```

**Особенности:**
- Формат времени: "21.88 с<br>16.45 м/с<br>59.232 км/ч"
- Лучшее время может быть выделено цветом (bgcolor=#ffd700)
- Статус "Отстранение" с colspan=6 вместо времени
- Категория "Неприбывшие участники" внизу таблицы

**Требуемые действия:**
1. Проверить текущий парсер `parse-results-racing.mjs`
2. Убедиться, что он корректно обрабатывает:
   - Формат времени/скорости
   - До 3 забегов
   - Статусы (Отстранение, Неявка)
   - Цвета для лучшего времени
3. Добавить тесты для разных форматов рейсинга
4. Проверить корректность извлечения судей

**Приоритет:** Высокий

### 11. Автоматическое обновление сайта для пользователей
**Проблема:** При деплое новой версии сайта пользователи могут видеть старую версию из-за кэширования браузера. Нет механизма автоматического обновления.

**Требование:** Обеспечить автоматическое обновление сайта у пользователей при наличии новой версии без необходимости вручную обновлять страницу.

**Возможные решения:**
1. **Service Worker (PWA):**
   - Проверять обновления при загрузке сайта
   - Уведомлять пользователя о наличии обновления
   - Автоматически обновлять при следующей загрузке
   - Использовать workbox для управления кэшем

2. **Cache-Control заголовки:**
   - Настроить правильные заголовки на Cloudflare Pages
   - Использовать ETag для валидации кэша
   - Установить короткое время кэширования для HTML
   - Долгое кэширование для статических файлов с версионированием

3. **Версионирование файлов:**
   - Vite уже использует хэши в именах файлов
   - Добавить версию в manifest.json для PWA
   - Использовать query parameters для принудительного обновления

4. **Автоматическое обновление страницы:**
   - Polling для проверки обновлений (раз в 5-10 минут)
   - WebSocket для мгновенного уведомления
   - Показывать баннер "Доступна новая версия" с кнопкой обновления

**Приоритет:** Средний

### 17. DogName — общий компонент для отображения клички
**Проблема:** Кличка собаки выводится минимум в трёх местах (таблица рейтинга, результаты события, профиль собаки) без единой логики форматирования.

**Текущий формат в БД:** `name_lat` (латиница) и `name_ru` (кириллица) — отдельные поля.

**Требование:** Создать компонент `DogName` (аналог `FiltersDropdown` — один компонент, используется везде):
- Основное имя: латиница крупно
- Если есть `name_ru` — русское имя ниже, тусклым цветом, без слэша
- Если есть только одно из двух — показывать только его, без пустых строк
- Если данные приходят одной строкой (формат "РУС / LAT" или "LAT / РУС") — функция разбивки по составу символов (кириллица vs латиница), **не по порядку слева/справа**, потому что порядок может меняться

**Места использования:**
- `frontend/src/components/DogStatsTable.tsx`
- `frontend/src/pages/Events/EventResults.tsx`
- `frontend/src/pages/DogProfile.tsx`
- Возможно `DogTooltip.tsx`

**Важно:** Сделать одним компонентом, не копировать логику в каждое место.

### 20. Ссылка на родословную (breedarchive.com)
**Описание:** Добавить возможность перейти на профиль собаки на breedarchive.com из рейтинга.

**Анализ:**
- Прямая ссылка на профиль автоматически **невозможна** — URL содержит случайный UUID из их базы, который нельзя вычислить из клички
- Пример: `https://saluki.breedarchive.com/animal/view/emul-de-gepard-gelila-al-rawda-699f7593-397c-4b71-89a7-09e352f30907`

**Рабочий компромисс:** Ссылка на поиск с предзаполненной латинской частью клички — человеку нужно лишь кликнуть на нужный результат поиска.

**Что нужно проверить перед реализацией:**
1. Точный параметр URL в форме поиска на `saluki.breedarchive.com/animal/index` (поле "Registered name") — нужно открыть страницу и посмотреть имя параметра в коде формы
2. Поддомены под каждую породу свои (`saluki.breedarchive.com`, `sloughi.breedarchive.com` — подтверждены). Нужно составить маппинг `порода → поддомен` и обрабатывать случай когда поддомена для породы нет (не показывать кнопку или показывать общую ссылку)

**Файл для реализации:** `frontend/src/pages/DogProfile.tsx`, возможно компонент `DogName` (пункт 17)
