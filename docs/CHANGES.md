# Изменения проекта ProCoursing Stats

## Дата: 23 июня 2026 (обновлено)

### Обзор
Исправлены критические проблемы безопасности, улучшена архитектура фронтенда, добавлена поддержка локальной разработки с mock-данными, загружены данные за 2015-2026 годы, реализована стратегия обновления данных.

---

## Критические исправления безопасности

### 1. SQL Injection в Worker (`src/worker.js`)
**Проблема:** Конкатенация строк в SQL запросах для параметров `breed` и `minStarts`.

**Исправление:** Параметризация всех SQL запросов через `bind()`:
- `/api/top/placement` - параметры теперь передаются через `bind(...params)`
- `/api/top/score` - аналогично параметризированы
- Добавлен `parseInt()` для `minStarts` для корректной типизации

**Файлы:**
- `src/worker.js` (lines 58-167)

---

## Улучшения UX

### 5. Разделение loading состояний (`frontend/src/pages/TopDogs.jsx`)
**Проблема:** Весь компонент показывал "Загрузка..." пока не загрузились все данные.

**Исправление:**
- Разделены состояния: `loadingFilters` и `loadingData`
- Фильтры загружаются отдельно и показываются сразу
- Таблицы загружаются независимо
- Добавлены отдельные индикаторы загрузки для каждого этапа

**Результат:** Улучшен UX - пользователь видит фильтры сразу, а таблицы загружаются асинхронно.

---

### 6. Обработка ошибок (`frontend/src/pages/TopDogs.jsx`)
**Проблема:** Ошибки только логировались в консоль, пользователь видел вечную "Загрузку...".

**Исправление:**
- Добавлено состояние `error`
- Отображение ошибки пользователю в красивом блоке
- Сообщение об использовании демо-данных при ошибке
- Fallback на mock-данные при недоступности API

---

### 7. Исправление надписи загрузки в Top Dogs
**Проблема:** При выборе породы надпись "Загрузка данных таблицы..." не пропадала.

**Исправление:** Добавлен useEffect для автоматической загрузки данных при изменении породы, убрано ручное управление loading состоянием.

---

## Исправления CORS

### 8. CORS проблема при локальной разработке
**Проблема:** Браузер блокировал запросы к `localhost:8787` из-за CORS.

**Исправление:** Изменен API URL с `localhost:8787` на `127.0.0.1:8787`:
- `frontend/src/services/api.js`
- `frontend/src/pages/Events.jsx`

**Причина:** Cloudflare Worker запускается на `127.0.0.1:8787`, а не `localhost:8787`.

## Стратегия обновления данных

### 11. Реализация гибридной стратегии обновления
**Созданные скрипты:**
- `scripts/backfill-historical.mjs` - бэкафлл исторических годов (один раз)
- `scripts/update-current-year.mjs` - инкрементальные обновления текущего года

**Изменения в БД:**
- Добавлена колонка `last_modified` в таблицу `events` для отслеживания изменений
- Применено к локальной базе данных

**Cron Trigger:**
- Реализован в `src/worker.js` с логированием
- Настроен на еженедельное выполнение (понедельник, 2 AM UTC) в `wrangler.toml`

**Текущее состояние:**
- Автоматическое обновление через Cron требует дополнительной настройки (внешний сервер или GitHub Actions)
- Для локальной разработки обновление выполняется вручную через скрипты

**Документация:** Подробная документация в `DATA_UPDATE_STRATEGY.md`
