## Обзор

## Основные изменения

### 1. Исправление отображения рейсинга в профиле собаки

**Проблема:** Данные рейсинга не отображались в профиле собаки "ГОЛДИ ГОЛДЕН ДАЛЕН" (ID: 680), несмотря на наличие записей в базе данных.

**Причина:** В таблице `results` записи рейсинга имели статус `'unknown_status_check_raw_text'` вместо `'finished'`. Backend API фильтровал результаты по условию `r.status = 'finished'`.

**Решение:** Обновили статус всех записей рейсинга с `'unknown_status_check_raw_text'` на `'finished'` в локальной и remote D1 базах данных.

**SQL команда:**
```sql
UPDATE results SET status = 'finished' WHERE status = 'unknown_status_check_raw_text'
```

**Файлы:**
- `backend/src/routes/dogs.ts` - проверено, что фильтр по статусу корректен

## Исправление парсера рейсинга

### 9. Поддержка двух форматов таблиц рейсинга

**Проблема:** Парсер рейсинга не работал для событий 2025 года из-за разницы в формате таблиц по сравнению с 2026 годом.

**Разница форматов:**
- **2025 (CC формат):** 18 ячеек, данные забегов в одной ячейке (время + скорость вместе)
- **2026 (ВС формат):** 21 ячейка, время и скорость в отдельных ячейках

**Изменение:** Обновлен `backend/parsers/parse-results-racing.ts` для автоматического определения формата таблицы по количеству ячеек.

**Добавленные функции:**
- `extractCellNumber()` - извлечение числа из ячейки (не только из `<i>`)
- `parseTime()` - извлечение времени из текста
- `parseSpeed()` - извлечение скорости из HTML

**Логика:**
- Если 18 ячеек → CC формат (2025)
- Если 21 ячейка → ВС формат (2026)
- Иначе → неизвестный формат (warning)

**Тестирование:**
- Создан тестовый скрипт `backend/scripts/test/test-racing-formats.ts`
- Созданы тестовые HTML файлы для обоих форматов
- Оба формата успешно парсятся

**Файлы:**
- `backend/parsers/parse-results-racing.ts` - обновлен для поддержки обоих форматов
- `backend/scripts/test/test-racing-formats.ts` - тестовый скрипт
- `backend/scripts/test/racing-2025-test.html` - тестовые данные формата 2025
