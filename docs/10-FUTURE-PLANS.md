## Оптимизация сборки

### 3. Разбиение больших файлов
**Проблема:** При сборке фронтенда появляется предупреждение:
```
dist/assets/index-B3yzHITO.js   652.47 kB │ gzip: 196.99 kB
(!) Some chunks are larger than 500 kB after minification
```

**Рекомендации Vite:**
- Использовать `dynamic import()` для code-splitting
- Настроить `build.rollupOptions.output.codeSplitting`
- Увеличить `build.chunkSizeWarningLimit`

**Возможные решения:**
1. **Code-splitting по страницам:**
   ```javascript
   const Judges = lazy(() => import('./pages/Judges'))
   const JudgeDetail = lazy(() => import('./pages/JudgeDetail'))
   const SpeedRecords = lazy(() => import('./pages/SpeedRecords'))
   // и т.д.
   ```

2. **Разделение компонентов:**
   - Вынести тяжелые компоненты в отдельные файлы
   - Использовать lazy loading для больших таблиц

3. **Настройка rollupOptions:**
   ```javascript
   build: {
     rollupOptions: {
       output: {
         manualChunks: {
           'react-vendor': ['react', 'react-dom', 'react-router-dom'],
           'ui-components': [...],
           'pages-judges': ['./pages/Judges', './pages/JudgeDetail'],
           'pages-speed': ['./pages/SpeedRecords', './pages/SpeedRecordsStats'],
         }
       }
     }
   }
   ```

**Приоритет:** Средний

## Парсинг результатов

### 4. Парсинг рейсинга (Racing)
**Проблема:** Формат результатов рейсинга существенно отличается от курсинга и БЗМП.

**Текущий формат рейсинга:**
- 18 колонок (vs 25 в курсинге/БЗМП)
- Нет rowspan=2
- Метрики: время и скорость (не судейские оценки)
- До 3 забегов на собаку
- Дистанция указана в метрах
- Попоны часто не указаны ("-")

**Пример структуры:**
```
Место | № | Порода | Класс | Пол | Кличка | Дистанция (м) | 
Забег 1 | Попона | Время 1 | Скорость | 
Забег 2 | Попона | Время 2 | Скорость |
Забег 3 | Попона | Время 3 | Скорость |
ВС | Титул(ы)
```

**Особенности:**
- Формат времени: "21.88 с<br>16.45 м/с<br>59.232 км/ч"
- Лучшее время может быть выделено цветом (bgcolor=#ffd700)
- Статус "Отстранение" с colspan=6 вместо времени
- Категория "Неприбывшие участники" внизу таблицы

**Требуемые действия:**
1. Проверить текущий парсер `parse-results-racing.mjs`
2. Убедиться, что он корректно обрабатывает:
   - Формат времени/скорости
   - До 3 забегов
   - Статусы (Отстранение, Неявка)
   - Цвета для лучшего времени
3. Добавить тесты для разных форматов рейсинга
4. Проверить корректность извлечения судей

**Приоритет:** Высокий
