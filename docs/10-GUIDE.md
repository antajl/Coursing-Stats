# Guide — Справочник `/guide`

Страница **Справочник** на сайте: правила курсинга/бегов, расшифровка протоколов, кратко о сайте.

**Код:** `frontend/src/pages/Guide/`

## Вкладки

| Вкладка | Файл | Содержание |
|---------|------|------------|
| Титулы и нормы | `TitlesTab.tsx` | CACL, CACIL, титулы ЧР/ПК/ГЧ РК, статусные соревнования |
| Протоколы | `ProtocolTab.tsx` | колонки протокола курсинга и бегов, квалификация ВС |
| О сайте | `SiteTab.tsx` | как устроен Coursing Stats, **индекс CS v1** (формула, примеры ИНГРИД 89,28 / контраст 88,85, эмпирика prior и курсинг vs БЗМП), контакты |

Переключение вкладок: `ToolbarSegmentControl` (как на других hub-страницах).

## Константы и тексты

**Файл:** `constants.ts`

- `COURSING_CERTIFICATES` — уровни сертификатов
- `COURSING_CACL_QUALIFICATION` — норма CACL (≥⅔ max баллов на статусных)
- `STATUS_EVENT_RULES` — правила второго места на статусных (без выдуманных «зазоров» от 1-го)
- `ABBREVIATIONS` — словарь для `AbbrTag`

## Компоненты UI

**Файл:** `components/GuideUi.tsx`

- `AbbrTag` — аббревиатура с tooltip из `ABBREVIATIONS`
- `SectionCard`, `InfoCallout` — карточки разделов

## Источники и правила редактирования

1. **Нормы РКФ** — только из официальных документов (правила курсинга, положение о титулах). При сомнении — сверка с PDF, не домыслы.
2. **CACL на 2-м месте** — положение п. 3.4.2, правила курсинга п. 4.3.4 (≥⅔ max).
3. **Нет официального правила** «зазора в очках от 1-го места» — из UI убрано намеренно.
4. **CC и ВС** — одна колонка квалификации, разные подписи в HTML протоколов (2025 vs 2026).
5. **Два рейтинга на сайте** (медали vs очки) — описывать во вкладке **«О сайте»**, не в «Титулы/Протоколы» (это не нормы РКФ). Для «по очкам» — **индекс CS v1** (`backend/lib/rating/coursing-rating-score.ts`): prior/k фиксированы, cap E=2, P только при B &gt; μ̃.
