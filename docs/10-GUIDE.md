# Guide — Справочник `/guide`

Страница **Справочник** на сайте: правила РКФ по **соревнованиям** (курсинг, бега) и **выставкам** (conformation), расшифровка протоколов, формула рейтинга CS, о проекте.

**Код:** `frontend/src/pages/Guide/`

**URL:** `/guide?tab=<id>` — вкладка сохраняется в query-параметре.

---

## Вкладки

| id | Подпись в UI | Файл | Содержание |
|----|--------------|------|------------|
| `titles` | Соревнования | `components/TitlesTab.tsx` | CACL, CACIL, титулы ЧР/ПК/ГЧ РК, статусные соревнования, особенности курсинга |
| `shows` | Выставки | `components/ShowsTab.tsx` | CAC, CACIB, BOB, BIS, ранги выставок, чемпионские титулы по красоте |

## Единая структура вкладок «Соревнования» и «Выставки»

Обе дисциплинные вкладки построены **одинаково по порядку блоков** — пользователь читает сверху вниз: общая карта → связь с сайтом → детали → справочник аббревиатур.

| # | Блок | Соревнования | Выставки |
|---|------|--------------|----------|
| 1 | Официальные источники | PDF/положения РКФ по курсингу и бегам | Положение о сертификатных выставках, титулах, FCI EXP-REG |
| 2 | **Иерархия** (3 уровня) | CACL → титулы старта → кумулятивные | CAC/CACIB → награды ринга → кумулятивные |
| 3 | **На этом сайте** | Раздел «Соревнования», два рейтинга | Раздел «Выставки», бейджи CAC/BOB/BIS в карточке |
| 4 | **Сертификаты** (сетка 3 кол.) | CACL / CACIL / CACLBr | CACIB / CAC / JCAC |
| 5 | **Награды на одном событии** | карточки EVENT_TITLES | карточки + нумерованный приоритет (BIS → … → R.CAC) |
| 6 | **Ранги: кто что получает** | таблица STATUS_EVENT_RULES | таблица SHOW_RANKS |
| 7 | **Кумулятивные титулы** | сетка 2 кол. (НЧ РК, ГЧР РК …) | сетка 2 кол. (C.I.B., Чемпион России …) |
| 8 | **Особенности дисциплины** | варианты оформления + COURSING_REQUIREMENTS | SHOW_FEATURE_NOTES (классы, оценки, ход дня) |
| 9 | **Сокращения** | таблица ABBREVIATIONS | таблица SHOW_ABBREVIATIONS |

**Иерархия наград выставок (один день, сверху престижнее):** BIS → BIG → BOB → BOS → CACIB → CAC → JCAC → CW → R.CAC/R.CACIB.  
**Бейджи в `ShowDogCard`:** BIS → BOB → CACIB → CAC (см. `frontend/src/pages/Shows/ShowDogCard.tsx`).

**Рейтинг выставочных собак:** индекс `rank_score` в `shows/indexes/dog-ranking.json`, сборка `backend/scripts/build-show-indexes.ts`, логика `backend/lib/show-award-ranking.ts`.

| Награда | Вес |
|---------|-----|
| BIS | 10 000 |
| BOB | 2 000 |
| CACIB | 500 |
| CAC | 100 |

Сортировка: `rank_score` ↓, затем счётчики по иерархии, затем `total_shows`. Поле `best_award` — лучшая категория награды (не место в классе). `best_placement` в индексе сохраняется, в UI не показывается.

**Парсинг титулов:** `parseShowTitles()` — CACIB не даёт лишний CAC; JCAC не считается как CAC.

**Кумулятивные титулы выставок (оформление в РКФ):** C.I.B. → Чемпион России → Юный чемпион → Чемпион РКФ → ветераны → КЧК.

| `NATIONAL_CHAMPION_VARIANTS`, `CH_RUSSIA_VARIANTS` | Варианты оформления титулов |

| `SHOW_EVENT_TITLES` | Карточки BIS, BOB, ЧРКФ, ЧФ |

| `SHOW_CHAMPIONSHIP_TITLES` | Кумулятивные титулы по красоте |

| `SHOW_FEATURE_NOTES` | Блок «Особенности выставок» |

## Источники и правила редактирования

### Соревнования (курсинг / бега)

1. **Нормы РКФ** — только из официальных документов (правила курсинга, положение о титулах). При сомнении — сверка с PDF, не домыслы.
2. **CACL на 2-м месте** — положение п. 3.4.2, правила курсинга п. 4.3.4 (≥⅔ max).
3. **Нет официального правила** «зазора в очках от 1-го места» — из UI убрано намеренно.
4. **CC и ВС** — одна колонка квалификации, разные подписи в HTML протоколов (2025 vs 2026).
5. **Два рейтинга на сайте** (медали vs очки) — вкладка **«Рейтинг»**, не «Соревнования». Формула CS v1 — `RatingTab.tsx`, код `backend/lib/rating/coursing-rating-score.ts`.

### Выставки (conformation)

1. **Положение о сертификатных выставках РКФ** (ред. 18.12.2024) — ранги, классы, CAC/CACIB, BOB, BIG, BIS.
2. **Положение о титулах РКФ** — оформление Чемпиона России, C.I.B., юных и ветеранов **по красоте** (не путать с рабочими титулами курсинга).
3. **CAC ≠ титул «Чемпион»** — сертификат на выставке; титул оформляется в РКФ по набору.
4. **speed_records / coursing_records** — не смешивать с выставками (отдельный раздел Донино).
5. При правке приоритетов — синхронизировать `SHOW_EVENT_AWARDS_PRIORITY` и порядок бейджей в `ShowDogCard`.

### Общее

- Вкладки **«Соревнования»** и **«Выставки»** — **разные дисциплины**; перекрёстные ссылки в intro-блоках.
- Не объединять рейтинги мест и очков; не делить `total_score` на число судей.
- Тексты справочника — **не** источник правды для парсеров; парсеры ориентируются на протоколы и PDF РКФ.
