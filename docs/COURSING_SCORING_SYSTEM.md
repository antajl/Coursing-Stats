# Система оценивания в курсинге

## Обзор

В курсинге используется система оценивания несколькими судьями по различным категориям. Результаты соревнований могут иметь разное количество судей (обычно 2 или 3), что создает проблему для сравнения результатов между разными соревнованиями.

## Структура таблицы результатов

Пример таблицы результатов соревнований (http://procoursing.ru/Complete_Results_2023-04-22.html):

| Колонка | Описание |
|---------|----------|
| 1 | Место |
| 2 | Номер (не важен) |
| 3 | Порода |
| 4 | Класс |
| 5 | Класс |
| 6 | Пол |
| 7 | Кличка |
| 8 | Забег 1 (номера и цвета) |
| 9-13 | Очки судей за забег 1 (3 колонки, по одной от каждого судьи) |
| 9 | Манёвренность |
| 10 | Скорость |
| 11 | Выносливость |
| 12 | Преследование |
| 13 | Энтузиазм |
| 14 | Сумма 1 (сумма очков всех судей за забег 1) |
| 15 | Забег 2 (номера и цвета) |
| 16-20 | Очки судей за забег 2 (аналогично забегу 1) |

| 24 | Титул |

## Категории оценивания

- **Манёвренность** (Ман.) - способность собаки маневрировать
- **Скорость** (Скор.) - скорость бега
- **Выносливость** (Вын.) - выносливость
- **Преследование** (Прес.) - качество преследования
- **Энтузиазм** (Энт.) - энтузиазм и желание работать

## Проблема сравнения результатов

На разных соревнованиях может быть разное количество судей:
- 2 судьи → сумма очков от 2 судей
- 3 судьи → сумма очков от 3 судей

Это делает невозможным прямое сравнение `total_score` между соревнованиями с разным количеством судей.

## Текущее состояние

В базе данных:
- `results.total_score` - хранит текущий итоговый балл (сумма всех очков)
- `results.raw_scores_json` - хранит исходные данные судейских баллов

Текущий формат `raw_scores_json`:
```json
{
  "heat1": [15,16,16,15,15,77],
  "sum1": 156,
  "heat2": [16,17,17,15,17,82],
  "sum2": 162
}
```

## Предлагаемое решение

Для честного сравнения результатов нужно:

1. **Нормализовать очки**: разделить сумму очков на количество судей, чтобы получить средний балл от одного судьи
2. **Хранить количество судей**: добавить поле `judge_count` в таблицу `results`
3. **Пересчитать total_score**: использовать нормализованный балл (средний от одного судьи)

### Пример

Соревнование с 3 судьями:
- Судья 1: 50 очков
- Судья 2: 48 очков  
- Судья 3: 52 очков
- Сумма: 150 очков
- Нормализованный балл: 150 / 3 = 50 очков

Соревнование с 2 судьями:
- Судья 1: 50 очков
- Судья 2: 48 очков
- Сумма: 98 очков
- Нормализованный балл: 98 / 2 = 49 очков

Теперь результаты сравнимы: 50 vs 49 очков.

## Требуемые изменения

1. Добавить поле `judge_count` в таблицу `results`
2. Обновить парсер для определения количества судей
3. Пересчитать `total_score` как нормализованный балл (сумма / количество судей)
4. Обновить все существующие записи в базе данных
