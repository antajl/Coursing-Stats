# Coursing Stats — промпт для Devin: редизайн UI

Скопируй этот документ целиком вместе с папкой `mockups/`:
`01-home.html`, `02-calendar-v2.html`, `03-rating.html`,
`04-dog-profile-additions.html`, `05-event-results-v2.html`,
`06-judges-v2.html`, `07-donino-v2.html`. Мокапы — статичный HTML/CSS,
референс структуры и настроения, а не код для копирования: реализуй их
заново как React-компоненты в текущей архитектуре, с реальными данными,
dark mode и доступностью.

Перед началом прочитай `docs/README.md`, `docs/00-PROJECT-STATUS.md`,
`docs/ai/GUIDELINES.md`, `docs/design/DESIGN-SYSTEM.md`,
`docs/architecture/API-REFERENCE.md`. Не нарушай зафиксированные там
архитектурные решения (два раздельных топа собак, весь архив 2015–2026,
`total_score` не делится на число судей, `/api/competitions`).

---

## 0. Главный принцип

Проект — уже функционально зрелый аналитический инструмент, а не набросок:
сортировка таблиц (Events, Judges, SpeedRecords), сворачиваемые строки
`<details>/<summary>` с медальными кружками и цветом попоны (EventResults),
3-уровневый drill-down судья → порода → собака → критерии (JudgeDetail),
экспорт карточки в PNG (DogProfile, SpeedRecords), проценты/ранг породы
(DogProfile), 3 вкладки рейтинга собак (по местам / по очкам / по скорости)
и 3 вкладки Донино (скорость / бега 350м / статистика). **Ничего из этого
не убирать и не упрощать.** Задача этой волны работ — типографика, отступы,
иконки, цвет, motion при разворачивании/переключении и несколько точечных
новых блоков там, где их реально не хватает (см. по каждой странице ниже).

Если в процессе кажется, что "было бы проще сделать плиткой вместо таблицы" —
это сигнал остановиться: почти наверняка таблица здесь даёт сортировку или
построчное сравнение, которое плитка на реальном объёме данных (сотни строк)
не даёт.

## ЧАСТЬ 1 — ФУНДАМЕНТ

### 1.1 Иконки

`lucide-react` уже в зависимостях, но не используется — заменить все ручные
inline `<svg>` в `App.tsx`, `FiltersDropdown.tsx`, `EmptyState.tsx`,
`ErrorState.tsx` на `lucide-react`, единый `strokeWidth={1.75}`. Создать
`frontend/src/lib/icons.ts` с картой смыслов → иконка (calendar, location,
club, judge, medal, speed, search, filters, chevronDown, close, menu, share,
sun, moon, externalLink, star/achievement, trendUp).

### 1.2 Силуэт собаки

Не тратить время на генерацию фото по каждой породе с последующей вырезкой
фона и трассировкой в SVG — дорого и даёт неровный результат. Сделать **один
универсальный силуэт бегущей борзой** (не по породам), переиспользуемый
как аватар-заглушка и декоративный мотив везде: в карточках рейтинга, в
профиле собаки, в hero главной страницы, в пустых состояниях. Если позже
понадобится точность по породам — взять готовый лицензионный lineart-набор
(Noun Project / Flaticon), не рисовать заново. Файл `DogSilhouettes.tsx`
сейчас содержит предупреждение "placeholder... need confirmation before
production use" в комментарии — заменить на финализированный силуэт и
убрать предупреждение.

### 1.3 Компонентная система

Создать отсутствующие сейчас переиспользуемые компоненты:
- `components/ui/Card.tsx`, `Button.tsx`, `Badge.tsx` (`tone: 'coursing' |
  'bzmp' | 'racing' | 'other' | 'gold' | 'neutral' | 'danger'`), `Chip.tsx`.
- `components/DogSilhouette.tsx` — см. 1.2.
- `components/DogName.tsx` — латиница жирным + кириллица под ней тусклее,
  использовать одинаково на TopDogs, DogProfile, EventResults, DogTooltip.
Пройтись по существующим страницам и заменить повторяющуюся ручную вёрстку
карточек/кнопок на эти компоненты, не трогая структуру таблиц/сортировки/
`<details>` — только классы оформления внутри них.

### 1.4 Типографика и спейсинг

Зафиксировать в `DESIGN-SYSTEM.md`: Montserrat — интерфейс, Georgia (serif) —
клички собак и крупные заголовки страниц/секций, JetBrains Mono — все числа
(баллы, скорость, время, места, счётчики). Спейсинг: `px-2.5 py-1` в мелких
бейджах/чипах, `p-4`/`p-5 md:p-6` в карточках, `gap-3.5`/`gap-4` между
карточками в сетке, `mt-8`/`mt-10` между крупными секциями страницы.

### 1.5 Motion — точечно

Не нужен масштабный анимационный слой поверх всего. Нужно:
- Плавный поворот шеврона при разворачивании `<details>` (CSS-переход
  `transform: rotate()` на `details[open] > summary svg`) — единообразно
  на всех уровнях вложенности в EventResults и JudgeDetail.
- Лёгкий `fade-in-up` при первой загрузке списка карточек только там, где
  список сам по себе карточный (главная, подиум/список TopDogs) — не в
  табличных списках (Events, Judges, SpeedRecords), где анимация появления
  мешает быстрому сканированию.
- Анимированный count-up для чисел на главной странице.
Все анимации короче 300ms, уважают `prefers-reduced-motion`.

### 1.6 Навигация

`App.tsx` Nav сейчас на трёх слоях `absolute`-позиционирования — переписать
на единый flex-контейнер (лого auto-слева / ссылки по центру или flex-1 /
действия auto-справа). Проверить на 768/820/900/1024/1280px. Дублирующийся
список "источники данных" в мобильном и десктопном меню — вынести в один
массив `NAV_SOURCES` и рендерить циклом в обоих местах.

### 2.2 Календарь событий — `mockups/02-calendar-v2.html`

**Файл:** `pages/Events/index.tsx`. Не заменять таблицу карточками —
при 200+ событиях крупная плитка даёт огромную высоту страницы и убивает
скорость сканирования. Решение — плотный список строк:

- Каждая строка — компактная (высота ~48-56px), на `grid-template-columns`,
  а не `<table>` и не крупная карточка. Сделать ОДНУ разметку строки,
  одинаково работающую на всех ширинах экрана — вместо текущих двух
  параллельных реализаций (desktop-таблица / mobile-карточки), которые
  рискуют разойтись при правках.
- Сортировка (по дате/виду/дисциплине/локации/судьям) — кнопка
  "Сортировка: Дата ▾" над списком, работает одинаково на всех размерах.
- Группировка по месяцам со sticky-заголовком месяца — даёт визуальные
  вехи при скролле длинного списка.
- Цвет дисциплины — тонкий (4px) левый бордер строки, не градиентная
  заливка всей строки (текущий градиент — источник визуального шума в
  тёмной теме).
- Чемпионаты/кубки России (`isImportantCompetition`) — золотой левый
  бордер + иконка-кубок + serif-начертание названия.
- Быстрые чипы-пресеты над существующим `FiltersDropdown` (не вместо него):
  "Ближайшие 30 дней", "Курсинг", "БЗМП", "Бега", "Чемпионаты и кубки".
- Легенда дисциплин — чипы с иконкой и подписью вместо кружочков.

### 2.4 Профиль собаки — `mockups/04-dog-profile-additions.html`
